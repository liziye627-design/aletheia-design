## AI驱动的多平台舆情真相验证系统

## 问题场景：信息过载与真相难辨

### 当今世界面临的挑战
- **信息爆炸**: 每天产生超过5亿条社交媒体内容
- **真假难辨**: AI生成内容、深度伪造、水军操纵泛滥
- **多平台碎片**: 信息散落在微博、抖音、Reddit等10+平台
- **时间紧迫**: 假新闻传播速度比真相快6倍

### 痛点
品牌公关危机、舆情监控滞后、假新闻泛滥、投资决策风险

---

## 解决方案：Aletheia三层架构

### Layer 1: 感知层 (Perception)
- **10大平台爬虫**: 微博、抖音、快手、Twitter、Reddit、豆瓣、知乎、B站、小红书、新闻聚合
- **实时数据采集**: 异步并发爬取，速率限制保护
- **多模态支持**: 文本、图片、视频、音频全覆盖

### Layer 2: 记忆层 (Memory) 
- **基准线建立**: 30天历史数据建立"正常状态"
- **异常检测**: Z-score检验识别舆情爆发
- **水军识别**: 批量账号、重复内容、0粉互动模式检测

### Layer 3: 推理层 (Reasoning)
- **多源验证**: 跨平台信息交叉对比
- **视觉分析**: Qwen2-VL-72B深度伪造检测
- **图像相似度**: 感知哈希算法识别PS/篡改
- **可信度评分**: 0-100分量化真实性

---

## 技术架构：企业级微服务设计

### 核心技术栈
- **后端**: FastAPI + AsyncIO异步框架
- **数据库**: PostgreSQL + TimescaleDB时序数据
- **缓存**: Redis分布式缓存
- **任务队列**: Celery + RabbitMQ异步任务
- **容器化**: Docker Compose一键部署
- **监控**: Prometheus + Grafana实时仪表板

### 稳定性保障
- **Circuit Breaker**: 熔断机制防止级联失败
- **Retry Strategy**: 指数退避重试策略
- **Rate Limiter**: 滑动窗口限流保护
- **Health Check**: 多层健康检查机制

### AI能力
- **SiliconFlow Qwen2-VL-72B**: 视觉内容分析
- **OpenCV + pHash**: 图像指纹识别
- **NER实体识别**: 提取关键人物/品牌/事件
- **情感分析**: 正/负/中性情感分类

## 核心功能展示

### 1. 多平台舆情监控
- **覆盖13+实际平台**: 中国(微博、抖音、快手、知乎、B站、小红书、豆瓣) + 国际(Twitter、Reddit、新闻聚合)
- **实时热搜追踪**: 并发抓取各平台热榜TOP50
- **关键词追踪**: 跨平台搜索品牌/人物/事件

### 2. 智能异常检测
- **提及量爆发**: 超过3σ标准差自动告警
- **水军识别**: 新账号占比异常、重复内容检测
- **账号类型分布**: 认证媒体/KOL/普通用户比例异常

### 3. 视觉真相验证
- **深度伪造检测**: AI分析图像是否PS/生成
- **图像溯源**: 感知哈希查找原图/重复图
- **OCR文字提取**: 图片中文字内容识别

### 4. 可信度评分引擎
- **多维度评估**:
  - 信息源可信度(30%)
  - 交叉验证一致性(25%)
  - 视觉真实性(20%)
  - 时序逻辑合理性(15%)
  - 社交信号质量(10%)
- **输出**: VERY_HIGH/HIGH/MEDIUM/LOW/VERY_LOW + 0-100分
