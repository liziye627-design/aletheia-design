# 某明星塌房事件多源验证

    platforms=["weibo", "douyin", "zhihu", "bilibili", "news"],
    limit_per_platform=100

# 结果分析
print(f"总采集: {result['summary']['total_posts']}条")
print(f"可信度: {result['credibility_score']:.2%}")

## 🔧 技术亮点

1. **统一数据标准化**: 所有平台数据统一为相同格式，便于跨平台分析
2. **并发采集优化**: 使用 `asyncio.gather` 并行抓取，性能提升3-5倍
3. **智能速率控制**: 每个爬虫独立速率限制，避免API封禁
4. **反爬机制应对**: 
   - 抖音: MD5签名
   - 小红书: 请求签名
   - B站: Cookie认证
5. **新闻源并行聚合**: 4家新闻媒体并行采集，2-3秒完成
6. **异常检测增强**: 跨平台Z-score对比，识别协同水军
7. **媒体类型丰富**: 支持图片/视频/文字/长文等多种内容类型

## 🚧 已知限制与解决方案

| 限制 | 影响 | 解决方案 |
|-----|------|---------|
| Twitter API限制 (15 req/15min) | 大规模采集受限 | 升级付费版或使用多账号 |
| 抖音签名算法复杂 | 可能失效 | 定期更新签名算法 |
| 小红书反爬较强 | 高频请求易封 | 使用Cookie池+代理IP |
| 知乎无Cookie限制多 | 部分内容无法获取 | 建议使用登录Cookie |
| 新闻聚合不支持评论 | 缺少用户反馈 | 补充社交平台数据 |
